# Skill: review-change

## Quando usar

Use esta skill para revisão transversal de uma entrega antes de considerá-la pronta.

## Leituras mínimas

- `.agents/templates/review-checklist.md`
- regras aplicáveis ao tipo de mudança
- arquivos alterados
- handoff da sessão, se existir

## Passos

1. Verificar fase ativa.
2. Revisar aderência arquitetural.
3. Revisar segurança.
4. Revisar validação executada.
5. Revisar documentação impactada.
6. Classificar lacunas como bloqueantes ou não bloqueantes.

## Saída esperada

- lista objetiva de conformidades;
- lista objetiva de lacunas;
- recomendação: pronto | pronto com ressalvas | não pronto.
